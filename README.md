# Осциллограф на STM32F411CEU6 (Black Pill)

Прошивка цифрового осциллографа на базе платы **Black Pill (STM32F411CEU6)** с
дисплеем **ILI9488 480×320** по SPI, захватом сигнала через **ADC1 + DMA**, двумя
энкодерами **EC11**, программным триггером и авто-измерениями.

Весь прикладной код (драйвер дисплея, шрифт 5×7, энкодеры, триггер, измерения,
UI, конечный автомат) написан с нуля. Для доступа к периферии используется
**только STM32 HAL**.

> ⚠️ В репозитории находится **только прикладной код** (`Core/Inc`, `Core/Src`).
> HAL/CMSIS и стартовый код добавляются через STM32CubeIDE/CubeMX (см. «Сборка»).

---

## Целевое железо

| Параметр | Значение |
|----------|----------|
| MCU      | STM32F411CEU6, Cortex-M4F, 100 МГц |
| Дисплей  | ILI9488 3.5", 480×320, SPI1 |
| Вход     | PA1 / ADC1_IN1 (аналоговый фронтенд, смещение 1.65 В) |
| Управление | 2× энкодер EC11 с кнопкой |

### Распиновка

```
SPI1 — дисплей ILI9488          ADC
  PA5  SPI1_SCK                   PA1  ADC1_IN1 (вход сигнала)
  PA6  SPI1_MISO
  PA7  SPI1_MOSI                Энкодер 1 (Time/div)
  PB0  LCD_DC                     PB3  ENC1_A
  PB1  LCD_CS  (active LOW)       PB4  ENC1_B
  PB2  LCD_RST (active LOW)       PB5  ENC1_SW (active LOW, pull-up)

                               Энкодер 2 (Volt/div)
                                 PB6  ENC2_A
                                 PB7  ENC2_B
                                 PB8  ENC2_SW (active LOW, pull-up)
```

## Структура проекта

```
Core/
├── Inc/   main.h oscilloscope.h ili9488.h adc_dma.h
│           encoder.h trigger.h display_ui.h measurements.h
└── Src/   main.c oscilloscope.c ili9488.c adc_dma.c
            encoder.c trigger.c display_ui.c measurements.c
Drivers/   (сюда CubeIDE положит STM32F4xx_HAL_Driver и CMSIS)
```

Порядок инициализации (`main.c`): `HAL_Init` → `SystemClock_Config` (100 МГц) →
GPIO → SPI1 → `ADC_DMA_Init` (ADC1+DMA2+TIM3) → `ILI9488_Init` → `Encoder_Init` →
`Trigger_Init` → `Oscilloscope_Init` → `while(1) Oscilloscope_Run()`.

## Сборка

### Вариант A — STM32CubeIDE (рекомендуется)

1. Создать новый проект **STM32 Project** для MCU `STM32F411CEU6`.
2. В `.ioc` минимально задать: HSE = Crystal/Ceramic Resonator (25 МГц), SYSCLK
   100 МГц, APB1 = 50 МГц, APB2 = 100 МГц. (Конфигурацию клоков всё равно задаёт
   `SystemClock_Config()` в `main.c`, поэтому периферию в CubeMX можно не
   настраивать.)
3. Сгенерировать проект — CubeIDE создаст `Drivers/STM32F4xx_HAL_Driver`,
   CMSIS, startup-файл и скрипт компоновщика.
4. **Заменить** сгенерированные `Core/Inc` и `Core/Src` файлами из этого репо
   (или скопировать поверх). Удалить из сгенерированного `stm32f4xx_it.c`
   дубликат `SysTick_Handler`, если возникнет конфликт (он определён в нашем
   `main.c`).
5. Убедиться, что в `stm32f4xx_hal_conf.h` включены модули HAL: `RCC`, `GPIO`,
   `SPI`, `ADC`, `DMA`, `TIM`, `CORTEX`, `PWR`.
6. Собрать. Должны быть включены флаги FPU:
   `-mcpu=cortex-m4 -mfpu=fpv4-sp-d16 -mfloat-abi=hard`.

### Вариант B — Makefile + arm-none-eabi-gcc

Понадобится `arm-none-eabi-gcc`, HAL/CMSIS от ST, startup `.s` и linker `.ld`
(их проще всего один раз сгенерировать CubeMX). Ключевые флаги:

```make
CFLAGS = -mcpu=cortex-m4 -mthumb -mfpu=fpv4-sp-d16 -mfloat-abi=hard \
         -DSTM32F411xE -std=c11 -O2 -Wall \
         -ICore/Inc -IDrivers/STM32F4xx_HAL_Driver/Inc \
         -IDrivers/CMSIS/Device/ST/STM32F4xx/Include -IDrivers/CMSIS/Include
LDFLAGS = -TSTM32F411CEUx_FLASH.ld --specs=nano.specs -lm
```

## Управление энкодерами

| Действие | Результат |
|----------|-----------|
| **ENC1 вращение** | Развёртка Time/div (быстрее/медленнее) |
| **ENC1 нажатие** (клик) | Режим триггера: AUTO → NORMAL → SINGLE → AUTO |
| **ENC1 удержание ≥0.5 c** | START / STOP захвата |
| **ENC2 вращение** | Масштаб Volt/div |
| **ENC2 вращение при зажатой кнопке** | Уровень триггера (шаг 0.05 В) |
| **ENC2 нажатие** (клик) | Фронт триггера: ↑ (Rising) / ↓ (Falling) |

В верхней строке экрана отображаются V/div, Time/div, режим/фронт триггера и
состояние RUN/STOP. В нижней строке — измерения Vpp, Freq, DC и уровень триггера.

## Калибровка (аттенюатор)

Прошивка пересчитывает напряжения исходя из:

```
V_pin   = raw / 4095 * 3.3 В          (напряжение на PA1)
V_вход  = (V_pin − 1.65) * G          (приведено ко входу)
```

где `G` — коэффициент аналогового фронтенда (аттенюатора):

- `osc.attenuator = 0` → **1:1**, `G = 1`
- `osc.attenuator = 1` → **1:10**, `G = 10`

Значение по умолчанию задаётся в `Oscilloscope_Init()` (`oscilloscope.c`).
Если коэффициент делителя на вашей плате иной, поправьте `osc_atten_gain()` в
`oscilloscope.c`. Смещение средней точки (1.65 В) задаётся константой
`ADC_OFFSET_V` в `Core/Inc/adc_dma.h` — подстройте под реальное смещение
фронтенда, если оно отличается.

Таблица чувствительности Volt/div и масштабные коэффициенты `scale`
(пиксели на отсчёт АЦП) — в `voltdiv_tbl[]` (`oscilloscope.c`):
`scale = GRID_DIV_Y * 3.3 / (4095 * V/div)`.

## Ограничения и заметки

- **Частота дискретизации.** Реальная развёртка ограничена максимальной частотой
  АЦП (≈ 2 МSa/s при времени выборки 3 цикла). Таблица `timebase_tbl[]` содержит
  и более быстрые подписи (1/2/5/10 мкс), но фактическая частота **ограничивается
  сверху** значением `OSC_MAX_SAMPLE_RATE` (2 МГц) в `oscilloscope.c`. Для самых
  быстрых градаций отображаемая развёртка будет «насыщена».
- **Полоса пропускания** определяется фронтендом и частотой дискретизации
  (теорема Котельникова): практически — единицы–сотни кГц в зависимости от Time/div.
- **Точность** ограничена 12-битным АЦП (±1 LSB ≈ 0.8 мВ на входе АЦП до
  аттенюатора) и точностью опорного напряжения 3.3 В. Для точных измерений
  опору и смещение 1.65 В стоит откалибровать.
- **CCMRAM и DMA (важно).** В ТЗ буферы АЦП просили разместить в CCMRAM, но на
  STM32F4 контроллеры **DMA не имеют доступа к CCMRAM**. Поэтому `adc_buf`
  размещён в обычном SRAM (`.bss`) — иначе захват не работал бы. См. комментарий
  в `adc_dma.c`.
- **Глубина цвета ILI9488.** Используется RGB565 (Interface Pixel Format `0x55`).
  Некоторые панели ILI9488 по 4-проводному SPI принимают только 18 бит/пиксель
  (`0x66`, 3 байта на пиксель). Если экран показывает искажённые цвета —
  переключите `PIXFMT` на `0x66` и адаптируйте `ili_push_color()` (3 байта на
  пиксель). Помечено `TODO` в `ili9488.c`.
- Метки `// TODO: проверить на железе` отмечают места, требующие подстройки
  (тайминги SPI/инициализации дисплея, время выборки АЦП vs импеданс источника).

## Карта модулей

| Модуль | Назначение |
|--------|------------|
| `ili9488.*`     | Драйвер дисплея, примитивы, шрифт 5×7, текст/числа |
| `adc_dma.*`     | ADC1+DMA2 (ping-pong), TIM3 как тактирование, модель ADC↔В |
| `encoder.*`     | Декодирование 2× EC11 (серый код + антидребезг) |
| `trigger.*`     | Поиск точки триггера по уровню/фронту |
| `measurements.*`| Vpp, Vmax/Vmin, DC, RMS, частота/период |
| `display_ui.*`  | Сетка, осциллограмма (частичная перерисовка), статус, метки |
| `oscilloscope.*`| Таблицы Time/Volt-div, конечный автомат, обработка энкодеров |
| `main.*`        | Тактирование 100 МГц, GPIO, SPI1, запуск суперцикла |
